# SDK 优化验收标准

> 本文用于验收后续 agent 的代码修改结果。本轮验收不包含 `CMakeLists.txt`。

## 1. 范围验收

必须满足：

- 没有重写算法公式。
- 没有修改串口协议解析逻辑。
- 没有把 OpenCV 引入 SDK 新增文件。
- 新增了 `matrix_hand_sdk.h` 和 `matrix_hand_sdk.cpp`。

检查命令：

```powershell
git diff --name-only
```

结果中不应出现：

```text
CMakeLists.txt
```

## 2. 公开头文件验收

检查文件：

```text
matrix_hand_sdk.h
```

必须满足：

- 包含 `MATRIX_HAND_CHANNEL_COUNT`，值为 19。
- 定义 `MatrixHandHandle`。
- 定义 `MatrixHandStatus`。
- 定义 `MatrixHandCalibrationStage`。
- 定义 `MatrixHandRuntimeConfig`。
- 定义 `MatrixHandAngleOutput`。
- 定义 `MatrixHandCalibrationResult`。
- 声明所有 SDK 函数。
- C++ 环境下有 `extern "C"`。
- 不包含 `hand_algorithm.h`。
- 不包含 `windows.h`。
- 不出现 `std::`。

检查命令：

```powershell
rg -n "hand_algorithm|windows.h|std::|vector|array|deque|string" matrix_hand_sdk.h
```

验收结果：

- 该命令不应命中任何内容。

## 3. SDK 函数验收

`matrix_hand_sdk.h` 必须声明：

```text
matrix_hand_create
matrix_hand_destroy
matrix_hand_reset
matrix_hand_set_config
matrix_hand_begin_calibration
matrix_hand_push_calibration_frame
matrix_hand_finish_calibration
matrix_hand_is_ready
matrix_hand_process_frame
matrix_hand_get_current_ad
matrix_hand_status_text
```

检查命令：

```powershell
rg -n "matrix_hand_create|matrix_hand_destroy|matrix_hand_reset|matrix_hand_set_config|matrix_hand_begin_calibration|matrix_hand_push_calibration_frame|matrix_hand_finish_calibration|matrix_hand_is_ready|matrix_hand_process_frame|matrix_hand_get_current_ad|matrix_hand_status_text" matrix_hand_sdk.h
```

验收结果：

- 每个函数名至少出现一次。

## 4. 错误码验收

必须定义以下错误码：

```text
MATRIX_HAND_OK
MATRIX_HAND_ERROR_NULL_HANDLE
MATRIX_HAND_ERROR_NULL_POINTER
MATRIX_HAND_ERROR_INVALID_CONFIG
MATRIX_HAND_ERROR_BAD_STATE
MATRIX_HAND_ERROR_NOT_READY
MATRIX_HAND_ERROR_CALIBRATION_NOT_ACTIVE
MATRIX_HAND_ERROR_CALIBRATION_EMPTY
MATRIX_HAND_ERROR_CALIBRATION_BAD_STAGE
MATRIX_HAND_ERROR_XTALK_FIT_FAILED
```

验收标准：

- 所有 SDK 函数除 `create/destroy/status_text` 外都返回 `int`。
- `matrix_hand_status_text()` 对每个错误码都有明确文本。
- 未知错误码返回 `"UNKNOWN_STATUS"`。

## 5. 生命周期验收

手工或测试代码验证以下链路：

```text
create
set_config
begin Closed
push Closed frame
finish Closed
begin Fist
push Fist frame
finish Fist
begin Spread
push Spread frame
finish Spread
is_ready == 1
process_frame
destroy
```

验收标准：

- 每一步返回 `MATRIX_HAND_OK`。
- Spread 完成后 `matrix_hand_is_ready()` 输出 1。
- `matrix_hand_process_frame()` 返回 `MATRIX_HAND_OK`。
- 输出角结构体不全为未初始化随机值。

## 6. 空指针验收

必须验证：

- 空 handle 调用返回 `MATRIX_HAND_ERROR_NULL_HANDLE`。
- 空 AD 指针返回 `MATRIX_HAND_ERROR_NULL_POINTER`。
- 空 output 指针返回 `MATRIX_HAND_ERROR_NULL_POINTER`。
- 空 result 指针返回 `MATRIX_HAND_ERROR_NULL_POINTER`。
- 空 ready 指针返回 `MATRIX_HAND_ERROR_NULL_POINTER`。

验收标准：

- 不崩溃。
- 不修改已有状态。

## 7. 阶段顺序验收

必须验证：

1. create 后直接 begin Fist。
2. create 后直接 begin Spread。
3. Closed 完成后直接 begin Spread。
4. Spread 未完成前 begin Crosstalk。

验收标准：

- 全部返回 `MATRIX_HAND_ERROR_CALIBRATION_BAD_STAGE` 或 `MATRIX_HAND_ERROR_BAD_STATE`。
- 错误调用不会推进 `completedStepCount`。
- 后续按正确顺序仍可完成校准。

## 8. 未 ready 处理验收

必须验证：

- create 后直接调用 `matrix_hand_process_frame()`。
- 只完成 Closed 后调用 `matrix_hand_process_frame()`。
- 只完成 Closed/Fist 后调用 `matrix_hand_process_frame()`。

验收标准：

- 返回 `MATRIX_HAND_ERROR_NOT_READY`。
- 输出结构体全部清零。

## 9. 配置状态验收

必须验证：

- create 后 set_config 成功。
- begin Calibration 后 set_config 返回 `MATRIX_HAND_ERROR_BAD_STATE`。
- Closed 完成后 set_config 返回 `MATRIX_HAND_ERROR_BAD_STATE`。
- Ready 后 set_config 返回 `MATRIX_HAND_ERROR_BAD_STATE`。
- 非法配置返回 `MATRIX_HAND_ERROR_INVALID_CONFIG`。

非法配置样例：

```text
mean_filter_window_frame_count = 0
thumb_gate_filter_window_size = 0
thumb_inward_gate_channel = 17
thumb_gate_deadband_ratio = -0.1
spread_deadband_ratio = 1.1
crosstalk_max_abs_intercept = -1
```

验收标准：

- 配置失败不清空已有校准状态。

## 10. 校准结果验收

必须验证 `MatrixHandCalibrationResult`：

- Closed 阶段：
  - `stage == MATRIX_HAND_CALIBRATION_CLOSED`
  - `frame_count > 0`
  - `invalid_channel_count == 0`
- Fist 阶段：
  - `stage == MATRIX_HAND_CALIBRATION_FIST`
  - 能报告跨度无效通道。
- Spread 阶段：
  - `stage == MATRIX_HAND_CALIBRATION_SPREAD`
  - 能报告展开通道无效。
- Crosstalk 阶段：
  - `stage == MATRIX_HAND_CALIBRATION_CROSSTALK`
  - 能填充 `xtalk_unstable_count` 和 `xtalk_unstable_channels`。

构造无效通道数据：

```text
Closed: 所有通道 = 1000
Fist: 某个弯曲通道仍为 1000
Spread: 某个展开通道仍为 1000
```

验收标准：

- 对应通道出现在 `invalid_channels`。
- 通道编号为 1-based。

## 11. 串扰失败验收

构造 Crosstalk 数据：

```text
所有帧完全相同
```

验收标准：

- `matrix_hand_finish_calibration()` 返回 `MATRIX_HAND_ERROR_XTALK_FIT_FAILED`。
- `HandAngleAlgorithm` 不启用串扰补偿。
- `getXtalkValidTargetChannelCount()` 返回 0。

## 12. OpenCV 依赖验收

检查命令：

```powershell
rg -n "opencv|OpenCV" matrix_hand_sdk.h matrix_hand_sdk.cpp hand_algorithm.h hand_algorithm.cpp README.md docs
```

验收标准：

- `matrix_hand_sdk.h`、`matrix_hand_sdk.cpp`、`hand_algorithm.h`、`hand_algorithm.cpp` 不应命中 OpenCV。
- README 不应把 OpenCV 描述为算法 SDK 依赖。

## 13. 串口隔离验收

检查命令：

```powershell
rg -n "SerialFrameReceiver|serial_port_io|ConnectionMode|windows.h|HANDLE|SetupDi|CreateFileW" matrix_hand_sdk.h matrix_hand_sdk.cpp
```

验收标准：

- 不应命中任何内容。
- SDK wrapper 不依赖串口层。

## 14. 文档验收

必须新增：

```text
docs/api.md
docs/calibration.md
```

`docs/api.md` 必须包含：

- handle 生命周期。
- 函数列表。
- 错误码表。
- 输出角字段说明。

`docs/calibration.md` 必须包含：

- 每阶段姿态说明。
- `MatrixHandCalibrationResult` 字段说明。
- 无效通道说明。
- 串扰异常通道说明。
